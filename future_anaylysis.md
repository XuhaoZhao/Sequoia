新增文件，实现期货价格K线的分析，第一步是使用这个方法获取期货合约列表 
contracts = db.get_active_main_contracts_simple(
            min_amount=20.0,          # 成交额>=20亿
            max_margin=50000.0,       # 保证金<5万
            max_fee=30.0,             # 手续费<30元
            use_volume_filter=True    # 启用成交额筛选
        )
 第二步是从future_5m表中，获取最新的1000条5 min K线，注意从future_5m是按月分表的，注意聚合的逻辑
future_5m中一般存储的是T-1或者夜盘的数据
第三步是从sina_futures_realtime中获取当日的实时数据，时间戳间隔时间是2s，你要把这个实时数据也聚合成5min 数据
第四步是把第三步和第二步的数据聚合起来，构成完整的5min K线数据
第五步 分析K线数据，计算前20根K线的收盘价和开盘价之间距离的绝对值，然后取平均值，然后计算最新K线的收盘价和开盘价之间的距离，计算最新距离和平均值的比值，对这个比值进行合约排名。
有一些点你要注意 ：（1）不同合约的最新K线数据时间戳可能不一致，5min时间点一过，你立马要把排名打印到控制台，这样我能看到及时的排名，其他时间不用打印，因为没有意义（2）当然有时候最新的数据没有到来，你可能要等一下最新的K线数据（3）上述逻辑要循环运行，你要决策一下计算频率，最新的实时K线数据是不断更新的（4）你要做好方法封装，写好注释，后续我可能自己修改代码
